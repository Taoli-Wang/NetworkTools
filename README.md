# NetworkTools
NetworkTools - 面向IT运维的轻量级网络测试工具集。集成ICMP Ping、TCP端口检测(TCPing)、路由追踪、子网计算、局域网扫描及IP地址管理（支持本地多方案与数据库同步）。绿色单文件免安装，Windows 10/11双击即用。
================================================================================
  Network Tools v4.6 使用说明
================================================================================

【 软件简介 】

  Network Tools 是一款面向 IT 运维人员的轻量级网络测试工具集，涵盖 ICMP
  Ping、TCP 端口检测、路由追踪、子网计算、局域网扫描、IP 地址批量管理等
  常用运维功能。纯绿色单文件，免安装，双击即用。


【 系统要求 】

  ⊳ Windows 10 / 11（64 位）
  ⊳ 无需安装 Python 或其他运行环境
  ⊳ 路由追踪功能需以管理员身份运行
  ⊳ 防火墙需放行 ICMP 协议（否则 Ping 功能不可用）

================================================================================
  三、各项功能详解
================================================================================

  1. 本机信息
     启动时自动获取并显示：
     · 计算机名（hostname）
     · 本机 IP 地址（局域网地址）
     · 公网出口 IP 地址（互联网出口 IP）
     底部 [复制本机信息] 按钮可一键将三行信息拷贝到剪贴板。

  2. 内网网关测试
     对 IP 地址管理中"内网网关"分组下的目标逐一执行 Ping 测试。

  3. 内网关键服务器测试
     对 IP 地址管理中"内网服务器"分组下的目标逐一执行 Ping 测试。

  4. 全部内网测试
     一键执行"内网网关"+"内网服务器"两项测试。

  5. 外网网络测试
     对 IP 地址管理中"外网地址"分组下的目标逐一执行 Ping 测试，
     用于判断互联网连通性（默认包含 baidu.com、google.com 等）。

  6. 运行全部测试
     一次性执行以上所有分组测试，结束后显示汇总统计（通/不通数量）。

  7. 长 Ping（持续 Ping）
     输入目标地址后持续发送 ICMP 包，实时观察延迟波动与丢包率，
     适用于判断网络稳定性。可随时停止。

  8. 路由追踪（Tracert）
     输入目标地址，逐跳显示数据包经过的路由节点及每跳延迟。
     需要管理员权限（右键 → 以管理员身份运行）。

  9. TCP 端口检测（TCPing）
     检测指定 IP 的指定端口是否可达，不依赖 ICMP 协议。
     适用于 ICMP 被防火墙屏蔽时仍能判断服务端口连通性。

  10. IP 地址管理（详见第四节）
      管理内网网关 / 内网服务器 / 外网地址三类目标列表。
      支持本地多套方案、数据库同步、CSV 导入导出。

  11. 网络计算器（详见第五节）
      输入 IP + 子网掩码，计算网络地址、广播地址、
      可用 IP 范围、可用主机数量。支持一键复制全部结果。

  12. 局域网扫描（详见第六节）
      扫描本机所在局域网段内的活跃设备，快速发现网内主机。
      80 线程并行扫描，/24 网段约 3~5 秒完成。

  13. 保存 / 清空日志
      将日志区输出保存为 .txt 文件；或一键清空当前日志。


================================================================================
  四、IP 地址管理（核心功能）
================================================================================

  IP 地址管理是工具的配置中心，所有测试目标的增删改都在此完成。

  【4.1 数据源切换】

    顶部 [本地内置] [数据库连接] 两个按钮用于切换数据源：
    · 本地内置 — 使用本地配置文件（config.json），可自由增删改
    · 数据库连接 — 从 SQL Server / MySQL 等数据库拉取 IP 列表
      数据库模式下数据为只读，点击编辑/删除会提示切换到本地模式

    切换后顶部会显示状态徽章：
    · [✓ 本地模式 · 方案名]（蓝色）
    · [🔒 数据库只读]（绿色）

  【4.2 多套方案管理（本地模式）】

    本地模式下顶部显示方案选择器：
    [默认方案 ▾]  [+]  [···]

    · 下拉框切换不同方案（默认方案 / 方案A / 方案B）
    · [+] 新增方案（输入方案名称）
    · [···] 重命名当前方案 / 删除当前方案
    · 不同方案的数据完全隔离，增删改仅影响当前选中方案
    · [保存配置] 保存当前方案数据到 config.json
    · [恢复默认] 仅重置当前选中方案为初始数据

    底部按钮栏（本地模式）：
    ┌──────────────────────────────────────────────────────┐
    │ [添加] [删除] [编辑] [上移] [下移]                     │
    │ [保存配置] [恢复默认] [导出CSV] [导入CSV]    [关闭]   │
    └──────────────────────────────────────────────────────┘

    IP 表格支持 Ctrl/Shift 多选行，[删除] 可批量操作。

  【4.3 数据库连接模式】

    切换到 [数据库连接] 后，方案选择器自动隐藏，顶部显示：
    [⚙ 配置]  [🔄 同步]  [📥 导入到本地]  [🗑 清除数据]

    · ⚙ 配置 — 打开数据库连接配置对话框（详见 4.4）
    · 🔄 同步 — 连接数据库，拉取 IP 数据到临时缓存
    · 📥 导入到本地 — 将数据库缓存数据合并到当前本地方案
      自动按 IP 去重，完成后切回本地模式，即可编辑
    · 🗑 清除数据 — 清除全部数据库缓存记录（避免配置残留）

  【4.4 数据库连接配置】

    打开"配置"对话框，填写以下参数：

    ODBC 驱动        默认 SQL Server（系统中已安装的 ODBC 驱动名）
    服务器/实例      数据库服务器地址（IP 或主机名）
    数据库           目标数据库名
    全部服务器表      存放 IP 数据的数据表名
    IP/域名列        表中的 IP 地址列名
    名称/备注列      表中的名称列名
    分组列           用于区分内网网关/内网服务器/外网地址的列名
                    （值为空时全部归入内网网关）

    认证方式：
    · Windows 身份验证（使用当前 Windows 账号连接）
    · SQL 身份验证（输入数据库账号密码）

    填写完成后点击"测试连接"验证配置是否正确。

  【4.5 CSV 导入 / 导出】

    CSV 文件格式（表头 + 数据行）：

      分组,IP/域名,名称/备注
      内网网关,192.168.1.1,主网关
      内网服务器,10.0.0.5,文件服务器
      外网地址,baidu.com,百度

    · 分组列必须为：内网网关 / 内网服务器 / 外网地址
    · 导入时自动跳过重复 IP 和无效格式
    · 推荐用 Excel 编辑后另存为 CSV (UTF-8) 格式


================================================================================
  五、网络计算器
================================================================================

  1. 在"IP 地址"框输入目标 IP（如 192.168.1.0）
  2. 在"子网掩码 / CIDR"框输入掩码，支持三种格式：
     · 点分十进制  255.255.255.0
     · CIDR 斜杠   /24
     · 纯数字      24
  3. 点击"计算"或按回车
  4. 结果显示：
     · 网络地址
     · 广播地址
     · 可用 IP 范围
     · 子网掩码（点分十进制 + CIDR）
     · 可用主机数量
  5. 底部 [一键复制结果] 将全部计算结果拷贝到剪贴板


================================================================================
  六、局域网扫描
================================================================================

  文件 → 局域网扫描（Ctrl+Shift+N）打开扫描窗口。

  软件自动识别本机网段（如 192.168.1.0/24），点击"开始扫描"
  后以 80 线程并行 Ping，几秒内完成。

  扫描结果以树形列表展示：
  · IP 地址
  · 主机名（如果能解析到）
  · 在线状态

  可随时点击"停止"中断扫描。

  九、常见问题

  Q: 点击"运行全部测试"后无反应？
  A: 请先打开"IP 地址管理"，在各分组中添加至少一个测试目标。

  Q: Ping 全部超时？
  A: 检查防火墙是否放行 ICMP 协议，或目标设备是否在线。

  Q: 外网 IP 显示为"无法获取"？
  A: 部分网络环境可能屏蔽了公网 IP 查询接口，不影响其他功能。

  Q: 路由追踪提示失败？
  A: 请右键 NetworkTools_Qt.exe → 以管理员身份运行。

  Q: 局域网扫描速度慢？
  A: v4.6 已优化为并行扫描，/24 网段约 3~5 秒完成。
     如果仍然很慢，检查防火墙规则。

  Q: 上传 Logo 后 exe 目录多出文件？
  A: v4.6 已修复。Logo 和配置文件全部存储在
     %APPDATA%/NetworkTools/，不会在 exe 目录生成文件。
